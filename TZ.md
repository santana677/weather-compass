# Техническое задание на проект: WeatherComfort

## 1. Цель проекта
Создание веб-сервиса для оценки комфортности погоды с целью планирования пеших прогулок. Сервис получает актуальные данные о погоде через OpenWeatherMap API, анализирует их с использованием математической модели и выдает рекомендацию — насколько приятно находиться на улице в ближайшие дни. Решение актуально для жителей городов, которые хотят планировать свой досуг с учётом погодных условий.

## 2. Роли пользователей
- **Гость (неавторизованный пользователь):** Может просматривать список городов, смотреть прогноз погоды на 5 дней, видеть индекс комфорта и график температуры. Регистрация не требуется для использования основного функционала.
- **Администратор:** Управляет городами через встроенную админку Django (добавление, редактирование, удаление городов). Также запускает команду обновления данных из внешнего API.


## 3. Модели данных (сущности)

### Модель 1: City (Город)
Хранит информацию о городах, для которых доступен прогноз.
- **name** (CharField) — название города
- **country** (CharField) — страна
- **api_id** (IntegerField, unique) — идентификатор города в OpenWeatherMap

### Модель 2: WeatherRecord (Запись погоды)
Хранит погодные данные для конкретного города на конкретную дату.
- **city** (ForeignKey → City) — город
- **date** (DateField) — дата прогноза
- **temp** (FloatField) — температура (°C)
- **humidity** (IntegerField) — влажность (%)
- **wind_speed** (FloatField) — скорость ветра (м/с)
- **description** (CharField) — текстовое описание погоды
- **unique_together** = ('city', 'date') — гарантирует уникальность записи

### Модель 3: ComfortRating (Оценка комфорта)
Хранит рассчитанный индекс комфорта для каждой записи погоды.
- **weather_record** (OneToOneField → WeatherRecord) — связь с записью погоды
- **city** (ForeignKey → City) — город (для быстрых фильтров)
- **date** (DateField) — дата
- **comfort_index** (FloatField) — числовой индекс комфорта
- **rating** (CharField, choices) — категория: excellent, good, average, poor

**Связи моделей:**
- City → WeatherRecord (один ко многим) — один город имеет много записей погоды
- WeatherRecord → ComfortRating (один к одному) — каждая запись погоды имеет одну оценку комфорта
- City → ComfortRating (один ко многим) — один город имеет много оценок комфорта

## 4. Ключевой функционал (User Stories)

1. **Просмотр списка городов:** Пользователь заходит на главную страницу и видит список всех доступных городов с их странами.
2. **Просмотр прогноза погоды:** Пользователь выбирает город и видит таблицу с прогнозом на 5 дней (температура, влажность, ветер, описание погоды).
3. **Оценка комфорта:** Для каждого дня в таблице отображается рассчитанный индекс комфорта с иконкой (☀️ Отлично, 😊 Хорошо, 🌤 Средне, 🌧 Плохо) и числовым значением.
4. **Визуализация данных:** На странице города автоматически генерируется график изменения температуры за 5 дней с помощью Matplotlib.
5. **Обновление данных:** Администратор запускает команду `python manage.py update_weather`, которая получает актуальный прогноз через OpenWeatherMap API и сохраняет его в базу данных.

## 5. Внешние интеграции и Аналитика

**Интеграция:**
- **OpenWeatherMap API** (бесплатный тариф Current Weather & 5-day forecast) — получение прогноза погоды для городов по их API ID.

**Аналитика:**
- **Расчет индекса комфорта** по формуле:  
  `comfort = (temp - 20) - (wind_speed / 10) + (humidity / 100)`  
  где результат интерпретируется как:
  - `> 5` → Отлично (excellent)
  - `0..5` → Хорошо (good)
  - `-5..0` → Средне (average)
  - `< -5` → Плохо (poor)

- **Визуализация** динамики температуры с помощью библиотеки **Matplotlib** (график с датами на оси X и температурой на оси Y).

- **Analytics:** Matplotlib
- **Frontend:** Bootstrap 5 (адаптивный дизайн)
